# NeuralDB — AI가 DB를 지켜주는 시스템

> 개발본부 여러분께 드리는 프로젝트 소개서

---

## 한 줄 요약

**"DB 장애가 터지기 전에, AI가 먼저 발견하고 알려주고 고쳐주는 시스템"**

---

## 왜 만들었나요?

우리가 겪는 DB 문제를 떠올려 보세요:

- 새벽 3시에 **"DB 느려요"** 전화가 옵니다
- 대시보드를 열어서 CPU, 메모리, 쿼리 로그를 하나하나 봅니다
- 원인을 찾는 데 30분, 조치하는 데 또 30분
- 같은 패턴의 장애가 다음 달에 또 발생합니다

NeuralDB는 이 과정을 **AI가 자동으로** 해줍니다.

---

## 어떻게 동작하나요?

### 1단계: 1초마다 DB를 체크합니다

기존 모니터링 도구는 10초~1분 간격으로 데이터를 수집합니다.
NeuralDB는 **1초 단위**로 수집해서, 순간적인 장애도 놓치지 않습니다.

```
기존 도구:  ──────────●──────────●──────────●──────────
            10초        10초        10초

NeuralDB:  ─●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●●─
            1초 1초 1초 ...
```

### 2단계: AI가 "이상하다"를 감지합니다

2주간 데이터를 학습해서 **"평소 패턴"** 을 기억합니다.
평일 오전에 CPU가 보통 40%인데, 갑자기 90%가 되면?
사람이 임계값을 설정하지 않아도 AI가 알아서 이상을 감지합니다.

### 3단계: "왜?"를 분석합니다

이상이 감지되면 AI가 자동으로 원인을 분석합니다:

- 과거 비슷한 장애 사례를 검색합니다 (RAG)
- 느린 쿼리, 인덱스 누락, Lock 경합 등을 동시에 분석합니다
- **"이 쿼리의 풀스캔이 원인이고, 인덱스를 추가하면 해결됩니다"** 같은 결론을 내립니다
- 신뢰도 점수(0~100%)를 함께 알려줘서, AI 판단을 얼마나 믿을 수 있는지 보여줍니다

### 4단계: 알려주거나, 직접 고칩니다

위험도에 따라 AI의 행동이 달라집니다:

| 위험도 | AI가 하는 일 | 예시 |
|--------|-------------|------|
| **안전** | 자동 실행 | ANALYZE, SELECT 조회 |
| **주의** | 승인 후 실행 | CREATE INDEX (인덱스 생성) |
| **위험** | 추천만 | VACUUM FULL, 세션 종료 |
| **치명** | 차단 | DROP TABLE, TRUNCATE |

## DBA Agent: 대화형 AI DBA

가장 강력한 기능은 **DBA Agent** 입니다.
채팅창에 한국어로 질문하면 AI가 실제 DB를 분석해서 답해줍니다.

```
나: "이 DB가 왜 느려?"

AI: [현황]
    - CPU 92%, Active Connections 187/200
    - 느린 쿼리 Top 1: SELECT * FROM orders WHERE user_id = ... (평균 3.2초)

    [원인 분석]
    - orders 테이블에 user_id 인덱스가 없어 Full Table Scan 발생
    - 87,000건 테이블을 매번 전체 스캔

    [권장 조치]
    - CREATE INDEX CONCURRENTLY idx_orders_user_id ON orders(user_id);
    - 예상 효과: 쿼리 시간 3.2초 → 0.05초
```

```
나: "인덱스 만들어줘"

AI: ⚠️ WARNING 등급 작업입니다.
    승인하시겠습니까? [실행] [취소]
```

## 기존 도구와 뭐가 다른가요?

| | Datadog | pganalyze | **NeuralDB** |
|---|---|---|---|
| 가격 | 월 수백만원 | 월 수십만원 | **자체 구축 (무료)** |
| 수집 주기 | 10초 | 10초~1분 | **1초** |
| AI 원인 분석 | 상관관계만 | 없음 | **RAG + 4-Head 동시 분석** |
| 자연어 질의 | 불가 | 불가 | **"테이블 목록 보여줘" → 결과** |
| 자동 조치 | 불가 | 인덱스 추천만 | **승인 후 자동 실행** |
| AI 대화 | 불가 | 불가 | **DBA Agent 채팅** |
| 설치 환경 | SaaS만 | SaaS | **온프레미스 / 클라우드** |
| 라이선스 | 상용 | 상용 | **오픈소스 (MIT/BSD)** |

## 기술적으로 어떻게 만들었나요?

관심 있는 분들을 위한 기술 스택입니다:

```
프론트엔드:  React + TypeScript + TailwindCSS + ECharts
백엔드:      Python + FastAPI + SQLAlchemy + Celery
데이터베이스: PostgreSQL 16 (메트릭 + 벡터 검색 통합)
캐시:        Valkey (Redis 호환, 오픈소스)
AI/LLM:     Claude / GPT / Ollama (온라인·오프라인 전환 가능)
벡터 검색:   pgvector (유사 장애 사례 검색)
```

- **52개 설계 문서** (Spec)로 모든 기능을 사전에 정의하고 구현
- **11개 아키텍처 결정 기록** (ADR)으로 "왜 이 기술을 선택했는지" 문서화
- **262개 인수 기준 테스트** 통과 (94% 커버리지)
