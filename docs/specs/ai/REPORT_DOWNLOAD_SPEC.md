## 메타데이터
- **Spec ID**: FS-AI-REPORT-002
- **PRD 참조**: FR-AI-005
- **우선순위**: P0 (MVP)
- **상태**: Approved
- **선행 Spec**: FS-AI-REPORT-001 (DBA 정기 리포트)

---

## 1. 개요

Celery Beat가 수행한 DBA 리포트를 DB에 영구 저장하고, 프론트엔드 Report 페이지에서 **목록 조회 + PDF 다운로드**를 제공합니다.

---

## 2. 데이터 모델

### `dba_reports` 테이블

```sql
CREATE TABLE dba_reports (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    instance_id UUID NOT NULL REFERENCES db_instances(id),
    instance_name VARCHAR(255) NOT NULL,
    period VARCHAR(10) NOT NULL,  -- daily | weekly | monthly
    start_at TIMESTAMPTZ NOT NULL,
    end_at TIMESTAMPTZ NOT NULL,
    report_data JSONB NOT NULL,   -- 전체 리포트 JSON
    ai_analysis TEXT NOT NULL DEFAULT '',
    incident_count INT NOT NULL DEFAULT 0,
    slow_query_count INT NOT NULL DEFAULT 0,
    slack_sent BOOLEAN NOT NULL DEFAULT FALSE,
    created_at TIMESTAMPTZ NOT NULL DEFAULT NOW()
);

CREATE INDEX ix_dba_reports_instance ON dba_reports(instance_id, created_at DESC);
CREATE INDEX ix_dba_reports_period ON dba_reports(period, created_at DESC);
```

## 3. API 엔드포인트

### 3.1 리포트 목록 조회
- **Method**: GET
- **Path**: `/api/v1/reports/dba`
- **Query**: `instance_id?`, `period?`, `limit=20`, `offset=0`
- **Response**: `{ items: DBAReportSummary[], total: int }`

```python
class DBAReportSummary(BaseModel):
    id: UUID
    instance_name: str
    period: str
    start_at: datetime
    end_at: datetime
    incident_count: int
    slow_query_count: int
    slack_sent: bool
    created_at: datetime
```

### 3.2 리포트 상세 조회
- **Method**: GET
- **Path**: `/api/v1/reports/dba/{report_id}`
- **Response**: 전체 리포트 JSON (report_data)

### 3.3 리포트 PDF 다운로드
- **Method**: GET
- **Path**: `/api/v1/reports/dba/{report_id}/pdf`
- **Response**: `application/pdf` (StreamingResponse)

### 3.4 리포트 수동 생성 (기존)
- **Method**: POST
- **Path**: `/api/v1/reports/dba`
- **변경**: 생성 후 DB에 저장, response에 `report_id` 추가

### 4.2 기술 선택: HTML → PDF (weasyprint)

- HTML 템플릿으로 리포트 렌더링 후 `weasyprint`로 PDF 변환
- 한국어 폰트 내장 (Noto Sans KR)
- 테이블, 차트 placeholder, 이모지 지원
